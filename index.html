Repository Management

The software for operation of an observatory requires a large number of packages, assuming that they aren't all lumped together in one huge repo. The SKA Github in South Africa illustrates this. It also illustrates how hard it is, from looking at the repo collection, what repos you would need for a specific purpose. As discussed further in Python Packages in Github, we use a naming scheme that reflects the directory structure of the Python packages. All the packages that are under Git management live in
/usr/local/lib/pythonX.Y/DSN-sci-packages
for Python packages. To make this work, there needs to be a file called DSN-Sci-packages.pth which contains the string used as the title of this list item.
/var/www/software/Python/DSN-Sci-packages/
for Git HTML pages associated with the packages. These pages live in the gh-pages branch of each package.
The trunk of the directory tree is contained in repos with a single name, such as MonitorControl and Electronics.
