Repository Management

The software for operation of an observatory requires a large number of packages, assuming that they aren't all lumped together in one huge repo. The SKA Github in South Africa illustrates this. It also illustrates how hard it is, from looking at the repo collection, what repos you would need for a specific purpose. As discussed further in Python Packages in Github, we use a naming scheme that reflects the directory structure of the Python packages. The trunk of the directory tree is contained in repos with a single name, such as MonitorControl and Electronics.

Repos for Top-level Directories

MonitorControl

This package provides Monitor and Control functions for DSN Radio Astronomy. It will replace much of what is now in site-packages/Observatory.

Electronics

This package provides functions for DSN operating standard instruments and their bus interfaces. It will replace code that is now also in site-packages/Observatory.
